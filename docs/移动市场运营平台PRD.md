## 2. 需求背景

### 2.1 业务背景

江西移动存量运营需围绕**中高端客户流失、流量运营、套外收入**等场景，实现从「指标异常发现 → 归因分析 → 客群圈选 → 策略制定 → 任务派发 → 效果评估」的闭环。现有运营多依赖分散报表与线下协同，存在：

- 指标异常与归因、客群、策略之间链路断裂；
- 地市维度异动识别与任务下发缺乏统一入口；
- 策略创建字段分散、渠道配置不灵活；
- 省—市—县—格—小区多级下钻与指标/调度未联动。

### 2.2 产品目标

1. 提供**指标树可视化洞察**，支持异常节点一键进入趋势、客群、根因、预警、调度。
2. 建立**四步根因分析画布**：异常识别 → 指标级归因 → 关联标签树 → 汇聚客群。
3. 支持**指标调度**与**策略调度**两条任务线，并在调度中心统一管理。
4. 策略创建页整合来源客群、策略信息、产品、渠道，支撑从根因/客群一键跳转创建。

### 2.3 目标用户

| 角色 | 典型诉求 |
|------|----------|
| 省公司存量运营 | 查看全省指标树、发起调度、审批策略 |
| 地市运营人员 | 接收指标调度任务、查看负责地市明细 |
| 策略运营 | 基于客群创建策略、配置产品与渠道 |
| 数据分析/洞察 | 查看根因、标签树、客群画像 |

### 2.4 范围说明

**本期 PRD 覆盖（已实现原型）：**

- 指标洞察与指标树详情
- 指标详情弹窗（趋势/客群/根因/预警）
- 根因分析四列画布与关联标签树
- 指标调度向导、策略创建、调度派发双 Hub
- 策略推荐/评估/分类等策略中心能力（基础）
- 流量运营首页、指标预警、客群洞察（关联入口）

**不在本期详细范围：** 占位页（任务归集、流程管理、行为洞察等）、真实后端接口与权限体系（原型以 localStorage/sessionStorage 模拟）。

## 4. 功能详述

### 4.1 指标洞察

#### 4.1.1 指标树列表

##### 4.1.1.1 功能详细说明

**页面路径：** `pages/indicator-insight.html`

**功能描述：**  
展示已配置指标树卡片列表，支持按名称、分类、异常状态筛选，可新增指标树并进入详情或编辑器。

**列表/筛选字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 指标树名称 | 输入框 | 模糊搜索 |
| 分类 | 下拉 | 流量/中高端/离网/家庭/国际等 |
| 指标状态 | 下拉 | 全部/有异常/无异常 |
| 卡片-名称 | 展示 | 指标树名称 |
| 卡片-分类 | 展示 | 业务分类 |
| 卡片-异常数 | 展示 | 异常节点数量 |
| 卡片-更新时间 | 展示 | 最近更新 |

**页面操作：**

| 操作 | 触发 | 结果 |
|------|------|------|
| 查询 | 点击「查询」 | 刷新卡片列表 |
| 重置 | 点击「重置」 | 清空筛选 |
| 新增指标树 | 点击按钮 | 打开弹窗，填写名称/分类/说明后创建 |
| 进入详情 | 点击卡片 | 跳转 `indicator-tree-detail.html?id={treeId}` |
| 编辑树 | 卡片内入口 | 跳转指标树编辑器 |

**交互逻辑：**

1. 用户从能力中心 → 指标洞察进入列表页。
2. 筛选后点击目标树卡片，带 `id` 参数进入详情。
3. 新增树仅在列表层维护元信息，结构编辑在编辑器完成。

##### 4.1.2.1 功能详细说明

**页面路径：** `pages/indicator-tree-detail.html`

**功能描述：**

| 节点 Hover 菜单 | 6 项操作 | 见下表 |

**节点 Hover 操作：**

| 操作 | 说明 |
|------|------|

3. Hover 节点显示浮层菜单，点击后打开弹窗或跳转，菜单点击不触发节点选中冲突。
4. 「编辑指标树」跳转 `indicator-tree-editor.html`。

### 4.2 根因分析

#### 4.2.1 四步分析画布

##### 4.2.1.1 功能详细说明

**实现文件：** `root-cause-flow.js`、`rc-tag-tree-ui.js`、`rc-tag-tree-data.js`

**功能描述：**  
对异常指标按归因路径横向展开四列：**异常识别 → 指标级归因 → 关联标签树 → 汇聚客群**。多路径时纵向分行，跨列 SVG 连线对齐。

**四列说明：**

| 列 | 步骤 | 内容 |
|----|------|------|
| 第1列 | 异常识别 | 异常指标卡：值、环比、检测来源（规则/模型）、详情文案 |
| 第2列 | 指标级归因 | 每条路径一张归因卡：路径序号、贡献度、子指标名/值/环比 |
| 第3列 | 关联标签树 | 每路径独立横向标签树，最多三级 |
| 第4列 | 汇聚客群 | 汇聚客群卡：规模、画像、创建策略 |

**第1列 - 异常识别卡字段：**

| 字段 | 说明 |
|------|------|
| 指标名称 | 异常根指标 |
| 指标值 | 当前值 |
| 环比 | mom |
| 检测来源 | 预警规则命中 / 模型检测 |
| 检测详情 | 规则或模型说明 |
| 操作 | 「查看异常分析报告」 |

**第2列 - 指标级归因卡字段：**

| 字段 | 说明 |
|------|------|
| 路径序号 | 路径 1、路径 2… |
| 贡献度 | 百分比 |
| 子指标名称 | 归因子指标 |
| 子指标值 | value |
| 环比 | mom |
| 连接点 | 右端口，连至标签树 |

**第3列 - 关联标签树：**

**标签卡字段：**

| 字段 | 说明 |
|------|------|
| 层级标识 | 一级/二级/三级标签 |
| 标签名称 | 标签名 |
| 取值类型 | 枚举 / 数值区间 |
| 枚举取值 | 一行：取值 + 客群人数 |
| 数值区间 | 区间展示如 (0~100) + 客群人数 |
| 连接点 | 左/右端口 |

**标签树规则：**

- 默认全部展开，无展开/收起按钮；
- 标签树内连线由 SVG 绘制；
- 跨列连线：异常→归因、归因→一级标签，首条一级标签与归因卡顶对齐。

**第4列 - 汇聚客群卡：**

| 字段 | 说明 |
|------|------|
| 客群名称 | 汇聚结果名称 |
| 客群规模 | scaleLabel |
| 操作 | 「客群画像分析」「创建营销策略」 |

**交互逻辑：**

1. 仅 `alert` 状态节点进入根因 Tab 时渲染完整画布。
2. 点击「查看异常分析报告」→ 打开 `#modal-anomaly-report`。
3. 点击「客群画像分析」→ 打开 `#modal-segment-city-portrait`（江西省地市维度）。
4. 点击「创建营销策略」→ 写入 session 上下文 → 跳转 `strategy-create.html`。
5. 画布整体可横向/纵向滚动；列宽随标签树最大宽度自适应。

##### 4.3.1.2 步骤一：选择区域

**统计卡片（4 项）：**

| 统计项 | 说明 |
|--------|------|
| 正常城市 | 非异动地市数 |
| 异动城市 | 异动地市数 |

| 未达成指标城市 | 达成进度 <100% |

**列表字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 勾选 | Checkbox | 选择待调度地市 |
| 地市 | 文本 | 11 地市名称 |
| 指标值 | 文本 | 该地市指标值 |
| 目标值 | 文本 | 分解目标 |
| 达成进度 | 进度条+% | 绿/橙/红 |

**操作：**

| 操作 | 说明 |

**交互逻辑：**

##### 4.3.1.3 步骤二：调度派发

**顶部区域：**

| 字段 | 类型 | 说明 |
|------|------|------|

**摘要行：** 指标名称、已选地市数

**每地市卡片（左右布局）：**

| 左侧 | 右侧 |
|------|------|

**负责人规则：**

- 若存在默认看管人：展示姓名、组织、角色，隐藏域提交；

**底部按钮：**

| 按钮 | 说明 |
|------|------|
| 上一步 | 返回步骤一，保留勾选与备注草稿 |

**交互逻辑：**

1. 步骤二内容区可纵向滚动，底部按钮固定可见。
2. 提交后生成任务 ID，confirm 是否跳转调度中心指标调度详情。
3. 数据持久化至 `localStorage.jxMetricDispatchRecords`。

### 4.4 策略中心

##### 4.4.1.2 第一节：来源客群（只读）

| 计划结束 | 是 | 日期 | |
| 策略目标与说明 | 是 | 多行文本 | KPI 与执行要求 |

---

##### 4.4.1.4 第三节：产品配置

| 字段 | 必填 | 说明 |
|------|------|------|

**已移除：** 附属产品

##### 4.4.1.5 第四节：渠道配置

**功能描述：** 支持多个渠道块，每块独立配置触达渠道及参数。

**单渠道块字段：**

| 字段 | 必填 | 说明 |
|------|------|------|

| 渠道参数 | 视渠道 | 见下表 |

**渠道参数（按渠道类型）：**

| 渠道 | 参数字段 |
|------|----------|
| 外呼 | 外呼话术模板、单日外呼上限、外呼时段 |
| 短信 | 短信模板、日发送上限 |
| APP | APP 展位、推送标题 |
| 厅店 | 厅店范围、店员任务包 |

**操作：**

| 操作 | 说明 |
|------|------|
| 添加渠道 | 底部按钮，新增一块渠道配置 |
| 删除 | 每块右上角，至少保留 1 块 |
| 切换渠道类型 | 刷新该块参数表单 |

**已移除：** 辅助渠道

---

##### 4.4.1.6 表单提交

**底部操作：**

| 按钮 | 说明 |
|------|------|
| 取消 / 返回列表 | 回策略推荐 |

3. 提交后写入 `localStorage.jxStrategyRecords`，并可选进入策略派发详情。

```text
IndicatorTree → TreeNode(alert) → RootCauseFlow

  → StrategyCreateContext → Strategy → Dispatch → Eval

TreeNode(alert) → MetricDispatch → regions[] + assignees[]
```

## 7. 验收标准（核心）

| 编号 | 验收项 | 通过标准 |
|------|--------|----------|
| AC-01 | 根因四列 | 异常节点展示四列，连线对齐，标签树三级可展示 |
