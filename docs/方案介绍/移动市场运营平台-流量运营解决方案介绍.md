## 一、方案总览

### 1.1 建设背景

在流量经营精细化运营背景下，省公司需要一套**从指标监测 → 异动发现 → 根因研判 → 任务调度 → 策略落地 → 效果评估**的闭环运营体系。传统模式下，指标分散在报表、客群分散在标签系统、策略分散在 IOP，运营人员需要在多个系统间反复切换，难以快速完成「发现问题—定位原因—组织协同—营销触达—复盘评估」的全链路动作。

本平台围绕**流量运营**场景，构建一体化运营工作台，帮助业务人员：

- **看得清**：按业务模块聚合核心流量指标，支持省→市→县→网格→小区五级地域穿透；
- **找得准**：通过指标树与 AI 根因分析，快速识别异动指标及传导链路、汇聚客群；
- **调得动**：将异动指标一键派发给地市负责人或跨部门主管，形成可跟踪的调度工单；
- **打得准**：基于根因汇聚客群或调度下发客群，快速创建精准营销策略；
- **评得好**：对策略落地活动进行触达与转化效果复盘。

### 1.2 业务闭环

```
流量运营概览（日常监测）
    ↓ 发现异动
指标洞察 → 流量运营指标树（深度分析）
    ↓ 根因研判
根因分析（传导链路 + 汇聚客群）
    ↓ 组织协同              ↓ 精准营销
指标调度 / 客群调度      营销策略创建
    ↓                        ↓
任务分类 / 调度派发        策略分类（审批 / 执行）
                              ↓
                         策略评估（效果复盘）
```

### 1.3 本期覆盖模块

| 序号 | 模块 | 定位 |
|------|------|------|
| 1 | 流量运营概览 | 流量指标日常监测看板 |
| 2 | 指标洞察 | 指标树目录与入口 |
| 3 | 流量运营指标树 | 单指标树可视化分析工作台 |
| 4 | 根因分析 | 异动指标 AI 根因研判 |
| 5 | 指标调度 | 异动指标跨地市/跨部门派发 |
| 6 | 策略分类（营销策略创建） | 策略全生命周期管理 |
| 7 | 策略评估 | 活动执行效果复盘 |

## 二、流量运营概览

### 2.1 页面介绍

**流量运营概览**是运营专区下的流量经营首页看板，面向省公司及地市流量运营人员，以模块化卡片形式集中展示流量规模、价值转化、客户与用量、来源分析等核心指标。

页面支持**数据月份**与**五级地域下钻**（省 → 市 → 县 → 网格 → 小区），切换地域后指标值联动刷新，帮助用户从全省视角快速定位到局部区域的指标表现。每张指标卡片展示主指标值、同比/环比、业务口径说明及近期趋势图，异动指标以醒目标识高亮，便于一线人员每日巡检。

该页面与**运营概览配置**能力联动，管理员可按业务需要自定义一级标题分组与指标清单，实现「配置一次、全专区生效」的灵活展示。

### 2.2 重点功能描述

| 功能 | 说明 |
|------|------|
| **模块化指标看板** | 按「核心指标」「主线指标」「负面清单」「重点关注指标」等业务模块分组展示，结构清晰、便于分工负责 |
| **五级地域穿透** | 顶栏支持选择数据月份及省/市/县/网格/小区，指标值随地域范围联动变化 |
| **指标趋势可视化** | 每张卡片内置近 6 期柱状图 + 环比折线，悬停可查看各期明细 |
| **异动高亮识别** | 触发预警规则的指标卡片自动标红，便于优先关注 |
| **业务口径说明** | 指标名称旁提供口径 tooltip，统一指标理解，减少沟通歧义 |
| **指标调度入口** | 异动或未达标指标卡片支持一键发起「指标调度」，直接进入派发流程 |
| **配置化展示** | 支持通过运营概览配置自定义专区指标模块，满足不同业务条线展示需求 |
| **空态友好提示** | 当月暂无数据的指标展示占位说明，避免误解为系统异常 |

### 2.3 核心功能操作流程

#### 流程一：日常指标巡检

1. 登录平台，进入 **运营专区 → 流量运营概览**。
2. 在顶栏 **地域下钻条** 选择当前关注范围（默认全省，可下钻至具体地市/区县）。
3. 选择 **数据月份**，确认查看账期。
4. 按模块浏览指标卡片，重点关注带异动标识的指标。
5. 悬停指标卡片查看 **趋势走势** 与 **业务口径**。
6. 对需跟进的异动指标，点击卡片 **「调度」** 按钮进入指标调度（见第五章）。

#### 流程二：地域下钻分析

1. 在顶栏地域选择器中，依次选择 **地市 → 区县 → 网格 → 小区**。
2. 观察指标卡片数值随地域缩小而精细化变化。
3. 定位到异动集中的地市或区县后，可进入 **指标洞察** 做进一步树形分析。

## 三、指标洞察

### 3.1 页面介绍

**指标洞察**是能力中心下洞察体系的指标树目录页，作为进入各类指标树详情分析的**统一入口**。页面以卡片形式展示已发布的指标树清单，包括指标树名称、所属分类、指标数量、最近更新时间，以及是否含有异动指标等信息。

用户可通过分类 Tab（全部 / 流量运营 / 存量运营 / 细分市场运营）与筛选条件（指标树名称、分类、指标状态）快速定位目标指标树。其中 **流量运营指标树** 涵盖 300+ 指标节点，是流量经营深度分析的核心载体。

### 3.2 重点功能描述

| 功能 | 说明 |
|------|------|
| **指标树目录管理** | 集中展示各业务域已发布指标树，避免在多个菜单中查找 |
| **分类 Tab 切换** | 按流量运营、存量运营等业务域快速过滤 |
| **多条件组合筛选** | 支持按名称、分类、是否含异动指标等条件检索 |
| **异动标识** | 含异动指标的指标树在卡片上醒目标识，便于优先处理 |
| **一键进入详情** | 点击卡片或「详情」按钮，跳转至对应指标树详情页 |
| **演示场景预置** | 支持按业务场景预置筛选条件，适配汇报演示 |

### 3.3 核心功能操作流程

#### 流程：从目录进入指标树分析

1. 进入 **能力中心 → 洞察中心 → 指标洞察**。
2. 点击 **「流量运营」** Tab，或输入指标树名称进行搜索。
3. 在卡片列表中识别带 **异动标识** 的「流量运营指标树」。
4. 点击 **「详情」**，进入指标树详情页（见第四章）。
5. 在详情页完成地域下钻、根因分析、调度等后续操作。

## 四、流量运营指标树

### 4.1 页面介绍

**流量运营指标树**（指标树详情页）是单棵指标树的可视化分析工作台。页面以**横向树形结构**展示指标层级关系，从顶层业务分类逐层下钻至叶子指标，每个指标节点展示当前值、同比、环比及异动状态。

页面集成**地域下钻条**、**指标悬浮操作菜单**、**地域分析抽屉**、**异动指标清单**、**调度记录**等能力，是流量运营人员开展深度分析、组织协同的核心作业页面。用户可在此完成「看指标 → 比地域 → 找根因 → 发调度 → 查记录」的完整分析闭环。

### 4.2 重点功能描述

| 功能 | 说明 |
|------|------|
| **横向指标树可视化** | 分类节点与指标节点分层展示，支持逐层展开/收起及一键全部展开 |
| **指标节点信息卡** | 展示指标值、同比、环比；异动节点红色高亮 |
| **五级地域联动** | 顶栏地域与账期切换后，树上指标值及地域抽屉数据同步刷新 |
| **悬浮快捷操作** | 鼠标悬停指标节点，弹出「加入地域分析」「指标调度」「客群调度」「根因分析」等菜单 |
| **地域分析抽屉** | 点击指标节点，右侧滑出各地市指标值、同比环比、目标达成进度对比表 |
| **地域分析购物车** | 支持将多个指标加入地域分析模板，进行跨指标横向对比 |
| **异动指标清单** | 一键汇总全树异动指标，展示关联客群规模估算，支持批量操作 |
| **调度记录** | 查看本指标树历史指标调度、客群调度、策略调度记录 |
| **客群调度** | 针对异动指标关联客群，省级按地市或地市按区县下钻，派发客群调度工单 |
| **根因分析入口** | 从悬停菜单、地域抽屉、异动列表等多处一键进入根因分析 |

### 4.3 核心功能操作流程

#### 流程一：指标树浏览与地域对比

1. 从指标洞察进入 **流量运营指标树** 详情页。
2. 在顶栏选择 **数据月份** 与 **地域范围**（如选择「赣州市」）。
3. 在横向树上 **展开** 关注的指标分支，浏览各层级指标表现。
4. **点击** 某一指标节点，右侧打开 **地域分析抽屉**，查看该指标在全省各地市的分布与达成情况。
5. 对需横向对比的多个指标，通过悬停菜单 **「加入地域分析」**，再点击右下角 FAB 进入全屏地域分析。

#### 流程二：异动指标处理

1. 点击页头 **「异动指标」** 按钮，打开异动指标清单弹窗。
2. 浏览异动指标列表（含指标值、环比、估算客群规模）。
3. 对单条异动指标，选择操作：
   - **根因分析** → 进入 AI 根因研判（见第五章）；
   - **指标调度** → 派发给地市/部门负责人（见第六章）；
   - **客群调度** → 按地市或区县拆分客群并派发（需先在树上选择地域范围）。
4. 处理完成后，可通过 **「调度记录」** 查看历史派发情况。

#### 流程三：客群调度（省级 / 地市级）

1. 在指标树上将地域下钻至 **全省**（省级调度）或 **某一地市**（区县级调度）。
2. 对 **异动指标** 悬停，点击 **「客群调度」**。
3. 进入客群调度向导：
   - **第一步**：查看异动客群总规模，勾选需调度的地市（省级）或区县（地市级）；
   - **第二步**：填写工单标题、派发说明、完成日期，为每个区域指派负责人及工单要求；
   - **确认派发** → 生成调度任务及子任务，同步写入客群管理。
4. 调度完成后，可在 **客群管理** 中查看来源为「客群调度」的客群，并在 **策略创建** 时选用。

## 五、根因分析

### 5.1 页面介绍

**根因分析**是嵌入在指标树详情页中的 AI 智能分析能力，针对**异动指标**自动构建「传导链路 → 异常指标 → 汇聚客群」的三列分析矩阵。系统基于指标关联关系与 SHAP 归因等方法，识别导致当前指标异动的上游传导因素，并将多条链路影响的客群汇聚为统一的「异动客群」，为后续调度与营销策略提供精准目标。

根因分析以全屏弹窗形式呈现，包含指标元信息、传导链路分支、异常指标结论、汇聚客群卡片，以及可下钻的**根因分析报告**、**链路 LLM 分析**、**画像分析**等辅助能力。

### 5.2 重点功能描述

| 功能 | 说明 |
|------|------|
| **三列根因矩阵** | 左列传导链路、中列异常指标、右列汇聚客群，结构清晰 |
| **多分支传导链路** | 展示多条可能的因果传导路径及贡献度 |
| **SHAP 归因分析** | 对异常指标进行可解释性归因，辅助业务理解 |
| **汇聚客群生成** | 将各链路客群合并为统一异动客群，展示规模与标签收敛 |
| **根因分析报告** | 输出完整分析报告，含影响范围、建议措施等 |
| **链路 LLM 分析** | 对单条传导链路进行自然语言解读 |
| **画像分析** | 查看链路关联客群的画像特征 |
| **动作闭环入口** | 汇聚客群支持「客群调度」「创建营销策略」一键跳转 |
| **正常指标空态** | 非异动指标展示「当前无异常」，避免无效分析 |

### 5.3 核心功能操作流程

#### 流程：异动指标根因研判至策略创建

1. 在 **流量运营指标树** 中，对异动指标节点悬停，点击 **「根因分析」**（或从异动列表、地域抽屉进入）。
2. 全屏打开根因分析弹窗，阅读指标 **元信息**（编码、口径、数据来源等）。
3. **左列**：逐条查看传导链路，点击 **「链路分析」** 获取 AI 解读，点击 **「画像分析」** 查看客群特征。
4. **中列**：阅读异常指标检测结论与 SHAP 归因结果。
5. **右列**：查看 **汇聚客群** 名称、规模及标签收敛情况。
6. 点击 **「查看根因分析报告」**，阅读完整报告（影响范围、因果推断、建议措施）。
7. 根据业务判断，选择后续动作：
   - **客群调度** → 按地市/区县拆分派发（见 4.3 流程三）；
   - **创建营销策略** → 携带汇聚客群上下文进入策略创建页（见 7.3 流程二）。

## 六、指标调度

### 6.1 页面介绍

**指标调度**是将异动或未达标指标**派发给责任人**的协同能力，支持 **跨地市调度** 与 **跨部门调度** 两种模式。跨地市模式按江西省 11 个地市拆分指标表现，勾选目标地市后为每个地市指派负责人并填写工单要求；跨部门模式面向省公司客户运营中心、市场部等部门主管派发协同任务。

指标调度可从 **流量运营概览**、**指标树详情** 等多处入口发起，派发后自动生成父任务与子任务，与 **任务分类**、**调度派发** 模块联动，实现「派发—认领—执行—反馈」的全流程跟踪。

### 6.2 重点功能描述

| 功能 | 说明 |
|------|------|
| **双模式调度** | 跨地市（按区域拆分）与跨部门（按组织协同） |
| **地市指标对比表** | 展示 11 地市指标值、同比、环比、目标达成进度、异动标识 |
| **批量勾选** | 支持「勾选全部异动」「勾选全部未达标」等快捷操作 |
| **表头排序** | 指标值、同比、环比、达成进度等列支持排序 |
| **工单信息填写** | 工单类型、标题、派发说明、完成日期（或时间） |
| **接收人指派** | 默认带出地市看管人，支持从组织架构树修改 |
| **子任务自动拆分** | 按勾选地市/部门自动生成子任务，便于分头跟进 |
| **任务状态跟踪** | 子任务状态含待认领、执行中、协同中、已完成等 |
| **调度记录沉淀** | 指标树详情「调度记录」可查看本树历史派发 |

### 6.3 核心功能操作流程

#### 流程一：跨地市指标调度

1. 在 **流量运营概览** 或 **指标树详情** 中，对目标指标点击 **「指标调度」**（或悬停菜单选择）。
2. 在弹窗 **第一步** 选择 **「跨地市调度」**，点击下一步。
3. **第二步**：在地市列表中勾选需派发的地市（可使用「勾选全部异动」快捷按钮），点击下一步。
4. **第三步**：填写 **工单标题**、**派发说明**、**完成日期**。
5. 在工单明细表中，为每个地市确认 **工单接收人**（可点击「修改」从组织架构选择），填写 **工单要求**。
6. 点击 **「确认派发」**，系统生成调度任务及各地市子任务。
7. 选择前往 **任务分类** 查看整体进度，或返回指标树继续分析。

#### 流程二：跨部门指标调度

1. 发起指标调度，在第一步选择 **「跨部门调度」**。
2. 按部门分组勾选需协同的 **部门主管**（如客户运营中心、市场部）。
3. 填写派发说明，确认派发。
4. 各部门主管在 **调度派发** 中认领任务，可进一步协同或部门内再派发。

## 七、策略分类与营销策略创建

### 7.1 策略分类 · 页面介绍

**策略分类**是策略中心的统一入口，按 **客群类策略**、**工单类策略**、**预警类策略** 三类组织策略资产。本期以 **客群类策略** 为核心，提供策略列表检索、状态管理、详情查看及创建入口。

列表展示策略编号、名称、执行周期、渠道、创建人、IOP 策略状态、IOP 同步状态等信息，支持按编号名称、创建人、投放周期、状态等条件筛选。用户可从本页进入 **营销策略创建**，或对已有策略执行提交审批、同步 IOP、修改、删除等操作。

### 7.2 策略分类 · 重点功能描述

| 功能 | 说明 |
|------|------|
| **三类策略 Tab** | 客群类 / 工单类 / 预警类分栏管理（本期客群类完整实现） |
| **多条件检索** | 编号名称、创建人、投放周期、IOP 状态、同步状态、归属渠道 |
| **IOP 生命周期** | 草稿、审批中、待执行、执行中、已完成等状态及对应操作 |
| **策略详情弹窗** | 查看策略配置要点、客群、产品、渠道等信息 |
| **创建入口** | 「创建营销策略」跳转至策略创建页 |
| **编辑与审批** | 支持草稿修改、提交审批、同步 IOP |

### 7.3 营销策略创建 · 页面介绍

**营销策略创建**是基于已选客群配置精准营销策略的作业页面，采用四步表单：**来源客群 → 策略基本信息 → 产品配置 → 渠道配置**。

客群可来自 **客群管理手动选择**、**根因分析汇聚客群跳转**、**客群调度下发客群** 等多种来源。创建页展示客群名称、客群来源、客群规模等只读信息，用户填写策略名称、投放周期、场景分类后，选择主产品及触达渠道（10085 外呼、中国移动 APP、营业厅网台、10086 呼入弹屏），配置各渠道参数，最终保存草稿或提交进入 IOP 审批流。

### 7.4 营销策略创建 · 重点功能描述

| 功能 | 说明 |
|------|------|
| **多来源客群接入** | 支持客群管理、根因分析、客群调度等来源，自动带入规模与关联指标 |
| **客群选择弹窗** | 可检索、筛选生效客群，展示来源标签与真实客群规模 |
| **策略基本信息** | 名称、投放周期、描述、策略类型、集团/运营场景、活动类型/目的 |
| **产品单选** | 从产品目录选择主推荐产品 |
| **四渠道配置** | 10085 外呼、中国移动 APP、营业厅网台、10086 呼入弹屏，各渠道独立参数 |
| **草稿与提交** | 支持保存草稿稍后编辑，或提交进入审批 |
| **编辑回填** | 通过 `?edit=` 参数加载已有策略进行修改 |

### 7.5 核心功能操作流程

#### 流程一：从策略分类创建营销策略

1. 进入 **能力中心 → 策略中心 → 策略分类**。
2. 确认在 **「客群类策略」** Tab，点击 **「创建营销策略」**。
3. 若未预填客群，在 **「选择客群」** 弹窗中检索并选择 **生效状态** 的客群（含 AI 根因生成、客群调度等来源）。
4. 确认客群后，页面展示 **客群名称**、**客群来源**、**客群规模**。
5. 填写 **策略基本信息**（名称、投放周期、策略类型、场景等必填项）。
6. 点击 **「选择产品」**，在产品弹窗中选定主产品并确认。
7. 点击 **「选择渠道」**，勾选触达渠道，在 Tab 面板中配置各渠道参数（外呼类型/话术、APP 运营位、网台运营位、弹屏模板等）。
8. 点击 **「保存草稿」** 或 **「提交」**，返回策略分类列表；提交后进入 IOP 审批流程。

#### 流程二：从根因分析创建营销策略

1. 在 **根因分析** 弹窗右列「汇聚客群」卡片上，点击 **「创建营销策略」**。
2. 系统自动携带汇聚客群名称、规模、关联指标等信息跳转至策略创建页。
3. 客群信息已预填，直接填写策略基本信息、选择产品与渠道。
4. 提交后，策略与根因分析客群形成关联，便于后续效果追溯。

#### 流程三：使用客群调度下发客群创建策略

1. 完成 **客群调度** 派发后，系统在 **客群管理** 中自动生成来源为「客群调度」的客群记录。
2. 进入 **策略分类 → 创建营销策略**，在客群选择弹窗中筛选或检索 **客群调度** 来源的客群。
3. 选择目标地市/区县客群（含真实规模），按流程一完成策略配置与提交。

## 八、策略评估

### 8.1 页面介绍

**策略评估**是评估中心下的活动执行效果复盘页面，用于查看营销策略落地后的**触达与转化表现**。页面以活动执行列表形式展示各营销活动的活动 ID、名称、执行地市/区县、触达渠道、总用户数、接触用户数、成功用户数及执行周期等信息。

用户可通过活动名称、渠道等条件筛选，对比不同活动、不同区域的执行效果，为策略优化与下一轮营销提供数据依据。页面与 **策略分类** 形成「策略配置 → 活动执行 → 效果评估」的闭环最后一环。

### 8.2 重点功能描述

| 功能 | 说明 |
|------|------|
| **活动执行列表** | 按活动维度汇总触达与成功用户数据 |
| **多维度筛选** | 支持按活动名称、渠道检索 |
| **地域维度展示** | 列表展示地市、区县，便于区域对比 |
| **四渠道覆盖** | 10085 外呼、中国移动 APP、营业厅网台、10086 呼入弹屏 |
| **核心效果指标** | 总用户、接触用户、成功用户及执行时间 |
| **分页浏览** | 支持大量活动记录的分页查看 |

### 8.3 核心功能操作流程

#### 流程：活动效果复盘

1. 进入 **能力中心 → 评估中心 → 策略评估**。
2. 在筛选区输入 **活动名称** 关键字，或选择 **渠道**（如 10085 外呼）。
3. 点击 **「查询」**，浏览匹配的活动执行记录。
4. 对比各活动的 **总用户**、**接触用户**、**成功用户**，评估触达率与转化率。
5. 结合执行周期，判断活动是否在有效期内达到预期。
6. 若需查看源策略配置，通过面包屑返回 **策略分类** 检索对应策略。

## 九、模块协同价值总结

| 业务场景 | 涉及模块 | 价值 |
|----------|----------|------|
| 每日流量指标巡检 | 流量运营概览 | 一屏掌握核心指标，异动早发现 |

| 策略全生命周期管理 | 策略分类 | 统一检索、审批、执行状态 |
| 营销效果复盘 | 策略评估 | 触达与转化数据支撑迭代优化 |
