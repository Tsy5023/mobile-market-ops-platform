# 移动市场运营平台 — 产品设计说明（原型阶段）

## 1. 用户体验分析

### 1.1 目标用户
| 角色 | 核心诉求 |
|------|----------|
| 省/市公司运营经理 | 掌握大盘指标、区域对比、异常预警 |
| 细分市场运营专员 | 流量/存量/国际等专题运营与任务执行 |
| 策略与数据分析师 | 洞察、策略推荐、效果评估与优化 |
| 调度与一线执行人员 | 任务接收、派发、跟踪、协同闭环 |
| 系统管理员 | 指标、预警、工作台与知识库配置 |

### 1.2 主要功能域（来自功能清单）
1. **运营概览**：一屏总览 KPI 与健康度  
2. **运营专区**：流量、存量（离网/家庭/中高端）、国际市场  
3. **能力中心**：洞察 → 策略 → 调度 → 评估 闭环  
4. **基础功能**：工作台、监控预警、配置、知识库  

### 1.3 核心交互逻辑
```
总览发现问题 → 洞察下钻定位 → 策略制定/推荐 → 调度派发执行 → 评估反馈优化
                    ↑___________________________________|
```
- **顶部导航**：切换一级业务域（概览 / 专区 / 能力 / 基础）  
- **左侧菜单**：展开二级模块，点击进入三级功能页  
- **内容区**：指标卡片 + 图表 + 列表/配置，支撑演示与后续开发  
- **底部状态栏**：系统状态、数据 freshness、环境标识

## 2. 产品界面规划（信息架构）

```
移动市场运营平台
├── 运营概览
│   └── 运营概览（Dashboard）
├── 运营专区
│   ├── 流量运营 → 流量运营首页
│   ├── 存量运营 → 离网维系 / 家庭融合运营 / 中高端保有
│   └── 细分市场 → 国际市场
├── 能力中心
│   ├── 洞察中心（5）
│   ├── 策略中心（3）
│   ├── 调度中心（7）
│   └── 评估中心（2）
└── 基础功能
    ├── 运营工作台 → 工作台
    ├── 监控预警管理（3）
    ├── 配置管理（3）
    └── 知识库 → 知识库管理
```

共 **31** 个三级页面，与 Excel 功能清单一一对应。

## 3. 高保真 UI 设计规范

| 维度 | 说明 |
|------|------|
| 布局 | 顶栏 56px + 侧栏 240px + 内容区 + 底栏 36px |
| 主色 | `#2563eb` 企业蓝，顶栏深色渐变 |
| 组件 | 统计卡片、柱状图、筛选条、数据表、徽章状态 |
| 图片 | Unsplash 真实商务/数据场景图（非占位符） |
| 字体 | 微软雅黑 / Segoe UI，14px 正文 |

页面类型模板：
- **dashboard**：概览类（指标+趋势+区域表）  
- **chart**：洞察/评估类（双图表+明细）  
- **list**：运营/调度列表类（筛选+表格）  
- **config**：规则/配置类（表单+记录表）  
- **workbench**：个人工作台（待办+快捷入口）

## 4. HTML 原型实现说明

| 路径 | 说明 |
|------|------|
| `index.html` | 主入口，iframe 嵌入 `pages/*.html` |
| `pages/*.html` | 31 个独立页面，可单独打开演示 |
| `assets/css/common.css` | 全局样式 |
| `assets/js/nav-data.js` | 导航树 + 页面元数据 |
| `assets/js/layout.js` | 子页面顶栏/侧栏渲染；iframe 内自动隐藏外壳 |

**本地打开**：双击 `D:\cursor_demo\mobile-market-ops-platform\index.html`

**重新生成页面**：`node build-full.js`

## 5. 演示建议话术（对客户）

1. 从 **运营概览** 展示大盘与区域对比  
2. 进入 **运营专区 → 流量运营** 说明拉新转化  
3. 进入 **能力中心 → 洞察/策略/调度** 演示闭环  
4. 以 **工作台 + 预警** 收尾，体现日常办公与风控  

后续开发可在各页面预留的按钮、表格、表单区域接入真实 API。
